Table 3.1 Comparison of CNN, RNN, and LSTM
==========================================================================================

Algorithm: CNN
Main Data Type: Images / spatial data
Working Principle: Uses convolution filters to learn spatial features such as edges, textures, and patterns
Common Applications: Image classification, segmentation, restoration, enhancement
Strengths: Strong spatial feature extraction, efficient for images, preserves structure
Limitation: Needs training data and may miss long-range context without extra modules
Suitability for This Project: Highest - core methodology used in the project

Algorithm: RNN
Main Data Type: Sequential / time-series data
Working Principle: Uses recurrent hidden states to carry information from previous steps
Common Applications: Speech recognition, text modeling, video sequence analysis
Strengths: Captures temporal dependencies, useful for ordered data
Limitation: Struggles with long-term dependencies and training instability
Suitability for This Project: Low - not suitable for static image enhancement

Algorithm: LSTM
Main Data Type: Long sequential / time-series data
Working Principle: Improved RNN with memory cells and gates for long-term dependency learning
Common Applications: Language modeling, forecasting, sequential video analysis
Strengths: Better long-range memory than RNN, more stable on long sequences
Limitation: More complex and heavier than RNN, still sequence-focused
Suitability for This Project: Moderate only for future video-based extension, not needed for current static-image task
