PROJECT NOTES: HARDWARE REQUIREMENTS, SOFTWARE REQUIREMENTS, DATA COLLECTION, AND DATA PREPROCESSING
Project: Underwater Image Enhancement
Prepared from analysis of the current project folder

==================================================
1. HARDWARE REQUIREMENTS
==================================================

1.1 Minimum hardware requirements
- CPU: Any modern 64-bit processor capable of running Python and PyTorch
- RAM: At least 8 GB system memory
- Storage: At least 10-20 GB free disk space for dataset folders, checkpoints, logs, and results
- GPU: Optional for basic execution; training can run on CPU, but it will be slow

1.2 Recommended hardware requirements
- CPU: Intel Core i5/i7, AMD Ryzen 5/7, or equivalent
- RAM: 16 GB or more
- GPU: NVIDIA GPU with CUDA support for faster training and inference
- VRAM: 4 GB or more recommended for comfortable training with image size 256 and batch size 8
- Storage: SSD recommended for faster dataset loading and checkpoint writing

1.3 Hardware usage in this project
- The training and evaluation pipeline is GPU-aware and automatically checks whether CUDA is available
- If CUDA is available, the project uses the GPU for model training and inference
- If CUDA is not available, the project falls back to CPU execution

1.4 Why GPU is useful here
- The model is trained for multiple epochs on paired underwater image data
- Each batch performs forward propagation, loss calculation, backpropagation, and optimizer updates
- GPU acceleration significantly reduces training time compared to CPU-only execution

1.5 Hardware-related observations from the code
- Batch size is set to 8 by default
- Image size is set to 256 x 256
- Multiple workers can be used in the DataLoader
- Checkpoints and result figures are stored locally during training

1.6 Important note
- The exact physical hardware used by the author is not explicitly recorded in the project files
- Therefore, this document reports project-supported and recommended hardware rather than a claimed exact machine specification

3. DATA COLLECTION
==================================================

3.1 Main dataset used for training
- Dataset: EUVP (Enhancing Underwater Visual Perception)
- Training structure used in this project:
  - trainA: degraded underwater input images
  - trainB: corresponding enhanced/target images
  - validation: validation images used for no-reference validation and evaluation

3.2 EUVP role in this project
- EUVP is the primary supervised training dataset
- The model learns a mapping from degraded underwater images to improved target images using paired data
- The EUVP validation folder is used for no-reference validation through UCIQE and UIQM metrics

3.3 Dataset counts used in project reports and project logic
- EUVP TrainA / TrainB: 2185 paired images
- EUVP Validation: 130 images

3.4 Additional datasets used for evaluation
- UIEB
  - raw-890: 890 raw underwater images
  - reference-890: 890 reference images for paired evaluation
  - challenging-60: 60 no-reference challenging images
- RUIE
  - Used for cross-domain evaluation
  - Includes UCCS, UIQS, and UTTS subsets
  - Project results mention approximately 4230 images in total


3.5 Expected local dataset folder structure
- data/EUVP/Paired/underwater_scenes/trainA
- data/EUVP/Paired/underwater_scenes/trainB
- data/EUVP/Paired/underwater_scenes/validation
- data/UIEB/raw-890
- data/UIEB/reference-890
- data/UIEB/challenging-60
- data/RUIE

3.6 Data collection interpretation for report writing
- The project does not collect underwater images directly through hardware sensors
- Instead, it uses benchmark underwater datasets collected from publicly available research sources and organizes them locally under the data directory

==================================================
4. DATA PREPROCESSING
==================================================

4.1 Input validation
- The dataset loader first checks whether the input and target folders exist
- If a required directory is missing, the loader raises an error

4.2 Image file filtering
- Only supported image file types are loaded
- Supported extensions include:
  - .jpg
  - .jpeg
  - .png
  - .bmp
  - .tif
  - .tiff

4.3 Filename matching for paired training
- In paired mode, the loader reads files from the input folder and matches them against files in the target folder
- Only filenames present in both folders are kept for training
- Unmatched files are skipped

4.4 Image loading
- Each image is loaded using Pillow
- Images are converted to RGB format before further processing

4.5 Resizing
- All images are resized to a fixed spatial resolution
- Configured image size in this project: 256 x 256
- This ensures consistent tensor shape during training and evaluation

4.6 Tensor conversion
- After resizing, images are converted into tensors using torchvision transforms
- This converts image pixel values into a PyTorch-compatible format for the model

4.7 Data augmentation during training
- Applied only during training on paired EUVP data
- Current augmentation operations:
  - random horizontal flip
  - random vertical flip
  - color jitter on input image only

4.8 Color jitter details
- Brightness variation: 0.2
- Contrast variation: 0.2
- Saturation variation: 0.2
- Hue variation: 0.05

4.9 Why augmentation is used
- It increases diversity in the training set
- It improves model robustness to moderate variations in underwater illumination and appearance
- It helps reduce overfitting to a limited set of scene layouts

4.10 Target handling
- In paired training mode, both input and target images are returned
- In unpaired or evaluation mode, only input tensors are returned
- If an image cannot be loaded, the loader safely returns zero tensors instead of crashing

4.11 Validation preprocessing
- The EUVP validation folder is treated as input-only for no-reference validation in the main training pipeline
- Validation images are resized and converted to tensors, but no paired target is used there
- During training, validation quality is measured with UCIQE and UIQM rather than validation loss

4.12 Evaluation preprocessing
- EUVP evaluation:
  - resize to configured image size
  - convert to tensor
  - compute UCIQE and UIQM before and after enhancement
- UIEB paired evaluation:
  - resize to configured image size
  - convert to tensor
  - compute PSNR, SSIM, UCIQE, and UIQM
- UIEB challenging-60 evaluation:
  - no-reference evaluation using UCIQE and UIQM
- RUIE evaluation:
  - recursively scan all relevant subset folders
  - resize and convert each image
  - compute UCIQE and UIQM before and after enhancement

==================================================
5. TRAINING-RELATED PREPROCESSING AND SETTINGS
==================================================

5.1 Default training hyperparameters
- Image size: 256
- Batch size: 8
- Epochs: 100
- Learning rate: 0.0005
- Save checkpoint every: 10 epochs

5.2 Loss setup
- Combined training loss is used in the main training script:
  - L1 loss weight = 0.6
  - Edge loss weight = 0.3
  - Color loss weight = 0.1

5.3 Validation strategy in the main training pipeline
- The main training script performs validation every epoch on the EUVP validation folder
- Since this validation folder is input-only in the current pipeline, validation uses:
  - UCIQE
  - UIQM
- The learning-rate scheduler monitors validation UCIQE

5.4 Optimizer and scheduler
- Optimizer: Adam
- Scheduler: ReduceLROnPlateau

==================================================
6. SHORT REPORT-READY SUMMARY
==================================================

This project is a PyTorch-based underwater image enhancement system. It primarily trains on the paired EUVP dataset, where trainA contains degraded underwater images and trainB contains corresponding target images. The project also evaluates performance on EUVP validation, UIEB, and RUIE datasets. Preprocessing includes file filtering, RGB conversion, resizing to 256 x 256, tensor conversion, paired filename matching, and training-time augmentation through flips and color jitter. The software stack is built around Python, PyTorch, Torchvision, NumPy, Pillow, OpenCV, scikit-image, Matplotlib, and Pandas. The code supports both CPU and CUDA-enabled GPU execution, with GPU recommended for efficient training and evaluation.
